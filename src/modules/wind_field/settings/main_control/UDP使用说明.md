# UDP风扇控制使用说明 (新协议版本)

## 概述

风墙设置界面现已集成UDP风扇控制功能，可以将40x40的风扇网格数据通过UDP协议发送到100个独立的电驱板。

## 系统架构

### 硬件配置
- **中控机**: 192.168.1.101（以太网口：Realtek PCIe GbE Family Controller）
- **电驱板数量**: 100个（ID: 0-99）
- **每板风扇数**: 16个
- **风扇布局**: 4x4
- **IP地址范围**: 192.168.1.1 - 192.168.1.100
  - 电驱板ID 0 -> 192.168.1.1
  - 电驱板ID 99 -> 192.168.1.100
- **UDP端口**: 5001
- **链路数量**: 10个（每个链路10个板）

### 网络连接
```
中控机 (192.168.1.101) ← Realtek PCIe GbE Family Controller
    |
    |-- 交换机
          |
          |-- 链路0: 板0-9 (192.168.1.1-10)
          |-- 链路1: 板10-19 (192.168.1.11-20)
          |-- ...
          |-- 链路9: 板90-99 (192.168.1.91-100)
```

### 重要说明
- **必须确保UDP数据包从以太网口发出**，而非WiFi
- 程序启动时会自动绑定到192.168.1.101网口
- 如需更换网口IP，请修改`udp_fan_sender.py`中的`local_bind_ip`配置

## 新协议格式 (5字节包)

### 数据包结构
```
字节偏移 | 大小 | 类型        | 说明
----------|------|-------------|------
0         | 1    | uint8       | 链路ID (0-9)
1         | 1    | uint8       | 电驱板ID (0-9, 链内编号)
2         | 1    | uint8       | 风扇ID (0-15)
3         | 1    | uint8       | PWM低字节 (0-999)
4         | 1    | uint8       | PWM高字节 (0-999)
```

### 协议特点
- **包大小**: 5字节（比旧协议减少95%）
- **字节序**: Little-Endian
- **PWM精度**: 1000级 (0-999)
- **发送方式**: UDP单播
- **无校验**: 简化协议，提高速度

### IP映射规则
```
全局板ID = 链路ID × 10 + 链内板ID
IP地址 = 192.168.1.(1 + 全局板ID)

示例:
  板ID 0  (链路0, 板0) -> 192.168.1.1
  板ID 1  (链路0, 板1) -> 192.168.1.2
  板ID 9  (链路0, 板9) -> 192.168.1.10
  板ID 10 (链路1, 板0) -> 192.168.1.11
  ...
  板ID 99 (链路9, 板9) -> 192.168.1.100
```

## 网格到电驱板的映射

40x40的风扇网格分为10x10个模块，每个模块4x4对应一个电驱板的16个风扇：

```
网格布局 (40x40)
┌─────────────────────────────────────────────────────┐
│ 模块(9,0) │ 模块(9,1) │ ... │ 模块(9,9) │ 第9行 (顶部)
│  板90      │  板91      │     │ 板99      │
├─────────────────────────────────────────────────────┤
│    ...    │    ...    │     │    ...    │
├─────────────────────────────────────────────────────┤
│ 模块(0,0) │ 模块(0,1) │ ... │ 模块(0,9) │ 第0行 (底部)
│  板0       │  板1       │     │ 板9       │
└─────────────────────────────────────────────────────┘
```

**映射公式**: `全局板ID = 模块行号 × 10 + 模块列号`

**风扇索引**: `风扇ID = (行 % 4) × 4 + (列 % 4)`

## 数据包示例

### 示例1: 控制板0的风扇0，设置PWM=500
```
目标: 板ID 0 (链路0, 板0), 风扇ID 0
IP: 192.168.1.1:5001

数据包 (5字节):
00 00 00 F4 01 00
```

| 字段 | 值 | 说明 |
|------|-----|------|
| chain_id | 0x00 | 链路ID |
| board_id | 0x00 | 链内板ID |
| fan_id | 0x00 | 风扇ID |
| pwm低 | 0xF4 | 500 & 0xFF = 244 |
| pwm高 | 0x01 | 500 >> 8 = 1 |

### 示例2: 控制板99的风扇15，设置PWM=999
```
目标: 板ID 99 (链路9, 板9), 风扇ID 15
IP: 192.168.1.100:5001

数据包 (5字节):
09 09 0F E7 03 00
```

## 使用方法

### 1. 启用UDP发送

在风墙设置界面的菜单栏中，选择 **"UDP控制"** -> **"启用UDP发送"**

### 2. 自动发送模式

选择 **"UDP控制"** -> **"数据变化时自动发送"**（默认勾选）

启用后，每次修改风扇数据时都会自动发送到电驱板。

### 3. 播放时自动发送

应用函数后，点击播放按钮，每个时间点会自动发送UDP数据。

## PWM值转换

界面中的风扇速度值（0-100%）会自动转换为PWM值（0-999）：

```
PWM值 = 百分比 / 100 × 999
```

例如：
- 0% -> PWM 0
- 50% -> PWM 499
- 100% -> PWM 999

## 发送性能

- **数据包大小**: 5字节
- **风扇总数**: 1600个
- **总数据包**: 1600个/次
- **预期速度**: ~2-5ms完成全部发送（100个板×16风扇）

## 注意事项

1. **网络连接**: 确保中控机与交换机正确连接
2. **IP配置**: 确保电驱板的IP地址为 192.168.1.1-100
3. **端口开放**: 确保UDP端口5001未被占用
4. **链路配置**: 确保FPGA端正确配置了10个链路

## 故障排查

### 发送失败
- 检查网络连接
- 检查目标IP是否可达: `ping 192.168.1.1`
- 检查UDP端口5001是否开放

### 数据未更新
- 确认UDP发送已启用
- 确认自动发送已开启
- 查看系统信息面板的错误消息
