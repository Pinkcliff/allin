# UDP风扇控制使用说明

## 概述

风墙设置界面现已集成UDP风扇控制功能，可以将40x40的风扇网格数据通过UDP协议发送到100个独立的电驱板。

## 系统架构

### 硬件配置
- **电驱板数量**: 100个（ID: 0-99）
- **每板风扇数**: 16个
- **风扇布局**: 4x4
- **IP地址范围**: 192.168.1.101 - 192.168.1.200
  - 电驱板ID 0 -> 192.168.1.101
  - 电驱板ID 99 -> 192.168.1.200
- **UDP端口**: 5001

### 网络连接
```
中控机 (192.168.1.10)
    |
    |-- 交换机
          |
          |-- 电驱板0 (192.168.1.101) - 16个风扇
          |-- 电驱板1 (192.168.1.102) - 16个风扇
          |-- ...
          |-- 电驱板99 (192.168.1.200) - 16个风扇
```

## 映射关系

### 网格到电驱板的映射

40x40的风扇网格分为10x10个模块，每个模块4x4对应一个电驱板：

```
网格布局 (40x40)
┌─────────────────────────────────────────────────────┐
│ 模块(9,0) │ 模块(9,1) │ ... │ 模块(9,9) │ 第9行 (顶部)
│  电驱板90  │  电驱板91  │     │  电驱板99  │
├─────────────────────────────────────────────────────┤
│    ...    │    ...    │     │    ...    │
├─────────────────────────────────────────────────────┤
│ 模块(0,0) │ 模块(0,1) │ ... │ 模块(0,9) │ 第0行 (底部)
│  电驱板0   │  电驱板1   │     │  电驱板9   │
└─────────────────────────────────────────────────────┘
```

**映射公式**: `电驱板ID = 模块行号 × 10 + 模块列号`

### 数据包格式

每个电驱板接收一个100字节的UDP数据包：

```
协议头 (16字节):
  - magic (2): "FC"
  - board_id (1): 电驱板ID (0-99)
  - sequence_number (2): 序列号
  - timestamp (4): 时间戳(毫秒)
  - control_flags (2): 控制标志
  - reserved (5): 保留

控制数据 (80字节):
  - 16个风扇，每个5字节:
    - fan_id (1): 风扇ID (0-15)
    - pwm_level (2): PWM值 (0-999)
    - fan_flags (2): 风扇标志

校验和 (4字节): CRC32
```

## 使用方法

### 1. 启用UDP发送

在风墙设置界面的菜单栏中，选择 **"UDP控制"** -> **"启用UDP发送"**

### 2. 自动发送模式

选择 **"UDP控制"** -> **"数据变化时自动发送"**

启用后，每次修改风扇数据时都会自动发送到电驱板：
- 点选模式设置风扇速度
- 笔刷工具修改数据
- 函数生成器应用函数
- 全部清零操作

### 3. 手动发送

#### 发送所有数据
选择 **"UDP控制"** -> **"立即发送所有数据"**

这将把当前40x40网格的所有数据发送到100个电驱板。

#### 发送选中数据
1. 在界面上选中需要的风扇单元
2. 选择 **"UDP控制"** -> **"发送选中数据"**

这将只发送选中风扇对应的电驱板数据。

### 4. 查看统计信息

选择 **"UDP控制"** -> **"UDP统计信息"**

显示：
- 总发送数据包数
- 成功发送数
- 失败发送数
- 成功率
- 最后发送时间

## PWM值转换

界面中的风扇速度值（0-100%）会自动转换为PWM值（0-999）：

```
PWM值 = 百分比 / 100 × 999
```

例如：
- 0% -> PWM 0
- 50% -> PWM 499
- 100% -> PWM 999

## 注意事项

1. **网络连接**: 确保中控机与交换机正确连接，交换机与所有电驱板正确连接

2. **IP配置**: 确保电驱板的IP地址配置正确（192.168.1.101-200）

3. **发送频率**: 启用自动发送时，频繁的数据更新可能导致大量UDP数据包，建议根据实际需求调整

4. **错误处理**: 如果发送失败，检查网络连接和目标设备是否在线

5. **关闭程序**: 程序关闭时会自动关闭UDP发送器，确保资源正确释放

## 故障排查

### 发送失败
1. 检查网络连接
2. 检查目标设备IP是否正确
3. 检查UDP端口5001是否被占用
4. 检查防火墙设置

### 数据未更新
1. 确认UDP发送已启用
2. 确认自动发送已开启（如需要）
3. 查看系统信息面板的错误消息

## 示例场景

### 场景1: 设置单个风扇
1. 切换到点选模式
2. 双击选择一个4x4模块（对应一个电驱板）
3. 在右侧面板设置速度（如50%）
4. 数据自动发送到对应电驱板（如启用自动发送）

### 场景2: 批量设置风扇
1. 切换到笔刷模式
2. 调整笔刷大小和数值
3. 在画布上拖动笔刷
4. 所有受影响的风扇数据自动发送（如启用自动发送）

### 场景3: 应用函数模式
1. 切换到函数模式
2. 选择函数类型（如径向函数）
3. 调整参数并应用
4. 函数生成的数据自动发送（如启用自动发送）
