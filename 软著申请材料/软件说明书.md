## 二、软件概述

### 2.1 软件背景

本软件是一款面向微小型无人机风洞测试的专业综合评估系统，旨在解决无人机风场环境测试中的多设备协同控制、数据采集与分析、可视化监控等关键问题。系统集成了风扇阵列控制、环境参数监测、动作捕捉数据采集、CFD前处理等多个功能模块，为无人机风洞测试提供一体化解决方案。

### 2.2 应用领域

- 无人机风洞试验测试
- 空气动力学研究
- 风场环境模拟与评估
- 多传感器数据融合分析
- 实验设备集成控制

### 2.3 主要用途

1. **风扇阵列控制**：控制1600台风扇生成各种风场模式
2. **环境参数监测**：实时采集温度、湿度、风速、压力等传感器数据
3. **动作捕捉集成**：接收和处理动捕系统数据，可视化无人机姿态轨迹
4. **数据管理分析**：提供历史数据查询、统计分析、数据导出功能
5. **CFD前处理**：为计算流体力学仿真生成网格和场景文件
6. **系统监控**：统一监控各子系统运行状态

## 三、功能说明

### 3.1 系统架构

本软件采用模块化设计，主要包含以下子系统：

```
┌─────────────────────────────────────────────────────────────┐
│                    融合平台主入口                            │
├─────────────────────────────────────────────────────────────┤
│                                                               │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐      │
│  │全局监控仪表盘│  │传感器数据采集│  │风场编辑器    │      │
│  └──────────────┘  └──────────────┘  └──────────────┘      │
│                                                               │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐      │
│  │硬件控制模块  │  │CFD前处理模块 │  │动捕数据模块  │      │
│  └──────────────┘  └──────────────┘  └──────────────┘      │
│                                                               │
│  ┌─────────────────────────────────────────────────────┐    │
│  │              核心数据层 (Redis/MongoDB)              │    │
│  └─────────────────────────────────────────────────────┘    │
└─────────────────────────────────────────────────────────────┘
```

### 3.2 核心功能模块

#### 3.2.1 全局监控仪表盘

提供统一的系统监控界面，集成多个功能Dock窗口：

- **系统监控Dock**：显示CPU、内存、网络等系统资源使用情况
- **通讯状态Dock**：监控各通讯协议的连接状态（TCP/IP、EtherCAT、Modbus等）
- **环境监测Dock**：实时显示温度、湿度、压力、风速等环境参数
- **日志中心Dock**：系统运行日志的实时显示和过滤
- **动捕数据Dock**：显示动作捕捉系统的实时数据
- **风机状态Dock**：1600台风扇的PWM值监控和热力图显示

#### 3.2.2 风场编辑器

提供可视化的风场配置和编辑功能：

- **3D可视化编辑**：在3D场景中布置风速区域和目标点
- **表达式解析**：支持自定义数学表达式定义风速分布
- **模板库管理**：预设多种常用风场模式（层流、湍流、梯度风等）
- **时间轴控制**：支持风场随时间变化的动画编辑
- **实时预览**：实时查看风场矢量和风速分布

#### 3.2.3 硬件控制模块

实现对各类硬件设备的统一控制：

- **风扇阵列控制**：
  - 支持1600台风扇（40×40阵列）的独立PWM控制
  - 提供批量控制、区域控制、梯度控制等多种控制模式
  - 支持CSV格式控制序列导入和回放

- **Modbus设备控制**：
  - 支持俯仰伺服、造雨装置、喷雾装置等设备
  - 标准Modbus RTU/TCP协议

- **EtherCAT设备控制**：
  - 支持电驱、风速传感器、温度传感器等
  - 微秒级实时控制周期

#### 3.2.4 传感器数据采集

提供完整的传感器数据采集和管理功能：

- **多通道采集**：
  - 1600路风扇PWM值
  - 100路温度传感器（-20~80℃）
  - 100路风速传感器（0~30m/s）
  - 4路温湿度传感器
  - 1路大气压力传感器

- **数据存储**：
  - Redis实时数据缓存
  - MongoDB历史数据归档
  - CSV格式导出

- **数据查询与分析**：
  - 历史数据查询和图表显示
  - 统计分析（最大值、最小值、平均值、标准差等）
  - 直方图分布分析

#### 3.2.5 CFD前处理模块

为计算流体力学仿真提供前处理功能：

- **场景生成**：根据风场配置生成CFD仿真场景文件
- **网格生成**：自动生成计算域网格
- **边界条件设置**：自动设置入口、出口、壁面等边界条件
- **风扇模型映射**：将风扇阵列映射到CFD边界条件

#### 3.2.6 动捕数据可视化

处理和可视化动作捕捉数据：

- **CSV数据加载**：支持LuMo动捕系统导出的CSV格式
- **3D轨迹显示**：标记点的3D空间轨迹可视化
- **播放控制**：支持播放、暂停、停止、跳转功能
- **统计分析**：每个标记点的位置统计分析
- **数据导出**：支持导出处理后的数据

### 3.3 通讯协议支持

| 协议类型 | 应用场景 | 实时性 |
|---------|---------|--------|
| TCP/IP | 主控制器通信 | 毫秒级 |
| EtherCAT | 电驱、风速传感、温度传感 | 微秒级 |
| Modbus RTU | 俯仰伺服、造雨、喷雾 | 毫秒级 |
| API | 动捕系统、仿真系统 | 毫秒级 |

## 四、技术特点

### 4.1 技术架构

- **开发语言**：Python 3.8+
- **GUI框架**：PySide6 (Qt6)
- **数据可视化**：Matplotlib, PyQtGraph
- **数据存储**：Redis, MongoDB
- **通讯协议**：Socket, Modbus-tk, EtherCAT

### 4.2 关键技术

1. **多线程架构**：采用工作线程模式，保证UI响应流畅
2. **信号槽机制**：基于Qt信号槽实现模块间松耦合通信
3. **数据缓存策略**：Redis实时缓存 + MongoDB持久化的双层存储
4. **表达式解析引擎**：支持复杂数学表达式的动态解析和计算
5. **3D可视化技术**：基于OpenGL的高性能3D渲染

### 4.3 性能指标

| 指标 | 数值 |
|------|------|
| 数据采集频率 | 最高100Hz |
| 风扇响应时间 | <10ms |
| 3D渲染帧率 | >30fps |
| 支持传感器通道数 | >1800路 |
| 数据存储容量 | TB级 |

### 4.4 创新点

1. **一体化融合平台**：首次将风场控制、数据采集、动捕、CFD等功能集成于统一平台
2. **大规模阵列控制**：实现1600台风扇的独立精确控制
3. **自定义风场表达式**：支持用户通过数学表达式定义任意风场分布
4. **多源数据融合**：整合传感器、动捕、仿真等多源数据
5. **实时3D可视化**：提供直观的3D风场可视化编辑界面

## 五、操作说明

### 5.1 系统启动

1. 双击运行 `main.py` 启动融合平台
2. 在启动器界面选择要启动的模块：
   - 全局监控仪表盘
   - 融合系统（仪表盘+传感器数据）
   - 传感器数据采集系统

### 5.2 风场配置流程

1. 打开风场编辑器
2. 新建或加载风场配置
3. 在3D视图中添加风速区域
4. 设置区域参数（位置、大小、风速表达式）
5. 通过时间轴设置风场变化
6. 预览并保存配置
7. 导出控制序列到硬件控制模块

### 5.3 数据采集流程

1. 打开传感器数据采集模块
2. 配置采集参数（采样率、时长）
3. 启动采集任务
4. 实时监控采集进度
5. 采集完成后查看和导出数据
6. 可选：同步到MongoDB进行长期存储

### 5.4 硬件控制流程

1. 配置硬件通讯参数
2. 连接硬件设备
3. 加载风场配置或控制序列
4. 启动控制任务
5. 监控设备运行状态
6. 停止控制并断开连接
